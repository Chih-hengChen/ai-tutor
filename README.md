# AI Tutor Skill

自适应 AI 导师，让 Claude Code 成为你的私人编程教练。

通过"讲解 → 实践 → 验证"的闭环，确保你对每个知识点的**确定性掌握**。支持三种学习模式，自动识别、按需加载。

## 特性

### 三种学习模式

- **知识模式** — 学概念、理解原理，强制做题验证
- **项目模式** — 边做项目边学，引导实现而非给答案
- **源码阅读模式** — 扫描当前项目目录，生成针对代码库的教学大纲

### 学习科学

- **前置知识检测** — 学 React Hooks 前先检查闭包基础，基础不足自动补课
- **间隔复习** — 基于艾宾浩斯曲线，到期自动在课前弹出复习题
- **错误分级** — 区分概念性/细节/接近正确三种错误，精准对症
- **难度降级** — 连续失败自动拆分知识点，逐个击破

### 工程体验

- **断点续学** — 进度持久化到本地文件，关掉终端下次继续
- **环境自动检查** — 项目模式自动检测开发环境，未安装则给出安装指引
- **学习面板** — `/ai-tutor status` 查看 ASCII 进度条和统计
- **全局配置** — 严格等级、语气风格、可视化偏好，按你习惯来
- **随时退出** — 说"退出教学"即刻保存进度离开
- **一键重置** — `/ai-tutor reset [主题]` 推倒重来

### 可视化辅助

- Mermaid 流程图、时序图、架构图
- ASCII 结构图、流程图
- 对比表格、演进图

## 快速开始

### 学习命令

```
/ai-tutor 我想学 React Hooks          # 知识模式
/ai-tutor 学 Node.js 做一个聊天室      # 项目模式
/ai-tutor 帮我梳理当前项目的鉴权模块    # 源码阅读模式（需要当前目录有项目）
```

也可以直接说：

```
"帮我理解 Rust 所有权机制"
"用 React 写一个 Todo App"
"看一下这个项目的路由架构"
```

### 管理命令

```
/ai-tutor status              # 查看学习进度面板
/ai-tutor reset react-hooks   # 重置某个主题的记录
/ai-tutor reset --all         # 重置所有记录（需二次确认）
```

教学中随时说 **"退出教学"** 或 **"保存进度并离开"** 即可安全退出。

## 三种模式详解

### 知识模式

适用于学习概念、理解原理。

```
你: /ai-tutor 我想理解 React Hooks

AI Tutor:
  1. 前置知识检测："你了解 JS 闭包吗？"（检测前置基础）
  2. 评估主题规模 → 中型知识域
  3. 生成大纲，让你确认并标记已掌握的节点
  4. 逐节点讲解 → 每个节点配一道实践题验证
  5. 未通过 → 分级响应（概念性/细节/接近正确）
  6. 通过 → 更新进度，进入下一节点
  7. 模块完成 → 生成归档总结文档
```

**大型知识域**（React 源码、V8 引擎等）：先展示架构鸟瞰图，让你选焦点区域，各区域独立推进。

### 项目模式

适用于通过做完整项目来学技术。

```
你: /ai-tutor 学 Node.js 做一个聊天室

AI Tutor:
  1. 澄清需求和技术栈
  2. 环境自动检查 → "检测到 Node.js v20.x ✓"
  3. 按里程碑拆分（MVP → 核心功能 → 增强）
  4. 每个功能：精简知识铺垫 → 给任务要求（不给完整代码）→ 你自己写
  5. 运行验证："打开两个标签页，A 发消息，B 能看到"
  6. 卡住时渐进提示：先看报错 → 引导定位 → 最后才给代码片段
  7. 里程碑完成 → 生成项目文档和踩坑总结
```

### 源码阅读模式

适用于理解已有项目代码。

```
你: /ai-tutor 帮我梳理当前项目的登录鉴权模块

AI Tutor:
  1. 扫描项目目录、读取 package.json，识别技术栈
  2. 生成代码地图（ASCII/Mermaid 模块关系图）
  3. 按阅读顺序拆分节点（入口 → 中间件 → Token 签发 → 刷新机制）
  4. 引导你打开特定文件的特定行："打开 auth.ts，重点看 23-47 行"
  5. 考核基于实际代码："Token 过期是在哪一行触发刷新的？"
  6. 完成后生成调用链时序图和设计模式总结
```

## 间隔复习机制

基于艾宾浩斯遗忘曲线，自动在课前弹出复习题：

| mastery_level | 复习间隔 |
|---------------|---------|
| 1（刚通过） | 3 天后 |
| 2（复习 1 次） | 7 天后 |
| 3（复习 2 次） | 14 天后 |

复习通过 → level +1。复习未通过 → level 回到 1。

## 全局配置

编辑 `~/.claude/ai-tutor/config.yaml`：

```yaml
# 严格等级
# hard: 少提示、严格判定、不允许跳过
# normal: 平衡模式（默认）
# lenient: 多提示、宽松判定、允许跳过并标记待复习
strictness: normal

# 默认可视化工具
# mermaid: 生成 Mermaid 图表（推荐在 VS Code/Obsidian 中查看）
# ascii: 生成 ASCII 图表（纯终端友好）
visual_tool: mermaid

# 语气风格
# strict: 严谨教授，简短肯定，重点指错
# encouraging: 鼓励学长，肯定进步，缓解挫败感
tone: encouraging
```

## 数据存储

```
~/.claude/ai-tutor/
├── config.yaml                  # 全局配置
├── records/                     # 进度记录
│   ├── react-hooks.md
│   ├── nodejs-chatroom.md
│   └── project-auth-module.md
└── summaries/                   # 归档总结
    ├── react-hooks_hooks-basics.md
    └── nodejs-chatroom_milestone-1.md
```

- 关闭会话后进度不丢失
- 归档总结可在任意 Markdown 阅读器查看
- `/ai-tutor reset [主题]` 可清除对应记录

## 文件结构

```
ai-tutor/
├── SKILL.md              # 入口：配置加载 + 模式识别 + 特殊指令 + 间隔复习 + 共享约束
├── knowledge-mode.md     # 知识模式工作流（含前置知识检测、大型知识域策略）
├── project-mode.md       # 项目模式工作流（含环境自动检查）
├── codebase-mode.md      # 源码阅读模式工作流
├── visual-aids.md        # 可视化工具箱 + 学习面板模板
└── README.md             # 本文件
```

**按需加载：** 触发时只加载 SKILL.md，根据模式再读取对应子文件，不浪费上下文窗口。
