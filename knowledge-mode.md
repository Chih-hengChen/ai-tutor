# 知识模式工作流

本文件为知识模式的详细工作流。当 SKILL.md 判定进入知识模式后，读取本文件执行。

## 适用场景

- 学一个技术概念（"学 React Hooks"、"理解 Rust 所有权"）
- 学一门语言的核心机制（"学 TypeScript 泛型"）
- 任何以"理解、掌握"为目标的学习

## 大纲策略：按规模分级

生成大纲前，评估知识域的**规模和深度**，选择合适的策略：

### 小型知识域（5-10 个知识点）

单层模块结构即可。如"学 TypeScript 泛型"：

```
模块1: 泛型基础
  1.1 什么是泛型，为什么需要
  1.2 泛型函数
  1.3 泛型约束 (extends)
模块2: 泛型进阶
  2.1 泛型接口和类型
  2.2 条件类型
  2.3 实战：封装类型安全的数据结构
```

### 中型知识域（10-25 个知识点）

两层结构，每个模块有明确主题。如"学 Express 框架"：

```
模块1: 核心概念
  1.1 路由机制
  1.2 中间件原理
  1.3 请求/响应处理
模块2: 数据处理
  2.1 body 解析
  2.2 文件上传
  2.3 错误处理中间件
模块3: 工程化
  3.1 路由模块化
  3.2 环境配置
  3.3 项目结构最佳实践
```

### 大型知识域（25+ 知识点，如 React 源码、V8 引擎）

采用**分层聚焦策略**：

1. **先出总览图**：用 Mermaid 或 ASCII 画出整体架构的高层次鸟瞰图，让用户先有全貌认知。
2. **分区域划模块**：按子系统划分（如 React → Fiber | Reconciliation | Hooks | Scheduler | Event System）。
3. **让用户选焦点**：展示鸟瞰图后，让用户选择想先深入哪个区域。
4. **每个区域独立成大纲**：选定的区域内再按小型知识域的方式拆分。
5. **跨区域引用**：涉及其他区域时简要提及并标注"这在 XX 模块会深入讲"。

示例（React 源码）：

```
总览: React 架构鸟瞰（先展示架构图）

区域A: Fiber 架构（用户选择先学这个）
  A.1 Fiber 节点的数据结构
  A.2 Fiber 树的构建过程
  A.3 current 树与 workInProgress 树
  A.4 Fiber 的副作用标记系统

区域B: 调度机制（后续模块）
区域C: Hooks 实现
区域D: 协调算法(Reconciliation)
区域E: 事件系统
```

## Phase 1: 摸底与规划

1. **恢复检测：** 检查 `~/.claude/ai-tutor/records/` 下未完成的知识模式记录。
   - 有 → 展示进度，询问继续还是新主题。
   - 无 → 继续。

2. **前置知识检测 (Prerequisite Check)：**

   在生成大纲前，评估该主题需要的核心前置知识。如果用户的 records 中没有相关已掌握记录，则进行前置测试。

   执行步骤：
   - 识别该主题的 1-3 个核心前置知识（如 React Hooks 需要：JS 闭包、this 绑定、数组方法）
   - 出 1 道快速诊断题（不要多，1 道即可，控制在 1 分钟内能回答）
   - 用户答出 → 正常继续
   - 用户答不出 → 建议先补基础："建议先学一下 [前置知识]，它对理解 [当前主题] 很关键。要不要先花几分钟过一下？"
   - 用户选择补 → 快速讲解前置知识的核心要点（不超过 3 分钟），出 1 道题验证通过后回到主流程
   - 用户选择跳过 → 尊重用户选择，继续主流程（在后续讲解中遇到相关概念时顺带提及）

3. **快速摸底：** 了解用户背景和目标。
   - 1-2 个问题即可，不要做长问卷。
   - 根据回答调整起点。

4. **评估规模：** 根据主题判断是小型/中型/大型知识域，选择对应大纲策略。

5. **生成大纲：** 按选定策略生成，大型知识域要先出总览图让用户选焦点。

6. **展示确认：** 允许用户标记已掌握节点、调整范围。

7. **写入文件：** 按知识模式模板写入记录。

## Phase 2: 理论与实践

1. **状态检查：** 读取记录，找到第一个待处理节点，标记 `[学习中]`。

2. **讲解：** 精炼讲解核心原理。每次只讲 1 个最小知识点。
   - 根据知识点性质选择讲解形式（参考 `visual-aids.md`）：
     - 概念性知识 → 类比 + 表格
     - 流程性知识 → Mermaid 流程图/时序图
     - 结构性知识 → ASCII 图示
     - 对比性知识 → 对比表格

3. **强制考核：** 抛出一道实践题。题型按知识点性质选择：
   - 概念理解 → 情景分析题
   - 代码能力 → 编码题
   - 调试能力 → 调试题
   - 源码理解 → 阅读题
   严禁问"听懂了吗"。

## Phase 3: 确定性验证

严格评估用户答案，按错误类型分级响应：

| 错误类型 | 特征 | 策略 |
|----------|------|------|
| 概念性错误 | 思路方向错 | 换角度重讲，用可视化辅助 |
| 细节错误 | 思路对但实现有瑕疵 | 指出关键遗漏，给一个提示 |
| 接近正确 | 只有小瑕疵 | 给部分提示，允许微调后通过 |

- 未通过 → 出全新同类题，绝不直接给答案。
- 同一节点连续失败 3 次 → 拆分为更小子知识点。
- 通过 → 更新记录为 `[已掌握]`，同时更新该节点的 `last_tested_date` 为今天，`mastery_level` 设为 1。追加学习日志。

## Phase 4: 阶段总结与知识沉淀

1. **知识梳理：** 整理该阶段的核心概念，生成：
   - 核心知识点清单
   - 关键流程图（参考 `visual-aids.md`）
   - 关键代码片段及解释

2. **个性化点评：** 易错点汇总、强项、复习建议。

3. **写入文件：** `~/.claude/ai-tutor/summaries/[主题slug]_[模块slug].md`。

4. **大型知识域额外处理：** 完成一个区域后：
   - 回到总览图，标注已完成区域
   - 让用户选择下一个聚焦区域
   - 新区域开始前，简要回顾与已完成区域的关联

5. **节奏建议：** 完成一个模块后提示休息。
